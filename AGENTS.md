# AGENTS.md

## Массовое добавление участников в аналогичные чаты MAX

Этот репозиторий поддерживает **один основной способ** добавления участников:
- `npm run members:add -- ...`
- Скрипт: `scripts/add-chat-members-2026.js`

Другие подходы для массового добавления не используются.

## Обязательные условия

- В `.env` задан `BOT_TOKEN`.
- Бот является админом в целевом чате и имеет `add_remove_members`.
- Есть файл источника ID (`.json` с полем `ids` или `.txt`).

## Стандартная рабочая схема (как в текущих прогонах)

1. Подготовить файл источника участников, например:
- `data/official_max_member_ids.json`

2. Запускать приглашение волнами по 1000 участников:
- `--start 0 --count 1000 --invite-batch-size 10`
- затем `--start 1000 --count 1000 --invite-batch-size 10`
- затем `--start 2000 ...` и так далее до конца списка.

3. Тайминг обязателен:
- между патчами (батчами по `10`) выдерживать `3` секунды (`--pause-ms 3000`);
- между волнами по `1000` выдерживать `1` минуту.

4. После каждого запуска использовать файлы результата:
- `data/add_*_result.json`
- `data/still_missing_*.txt`

5. Повторные попытки по `still_missing` не делать.
- Никаких ретреев по недобавленным пользователям.
- `still_missing` сохранять только для отчёта и анализа, без повторного запуска.

6. Режим выполнения:
- работать нонстопом, пока не обработан весь список;
- не отправлять промежуточные комментарии;
- отправлять только итоговый отчёт по завершению.

## Базовые команды

```bash
# Первая волна
npm run members:add -- \
  --source-file data/SOURCE_MEMBER_IDS.json \
  --target-chat-id -12345678901234 \
  --start 0 \
  --count 1000 \
  --invite-batch-size 10 \
  --pause-ms 3000

# Следующая волна
npm run members:add -- \
  --source-file data/SOURCE_MEMBER_IDS.json \
  --target-chat-id -12345678901234 \
  --start 1000 \
  --count 1000 \
  --invite-batch-size 10 \
  --pause-ms 3000
```

## Правило поддержки

Если требуется массовое добавление в новый аналогичный чат, использовать только эту схему.

## Триггер продолжения

Кодовое слово: `код красный`

Поведение:
- Если ранее запуск `npm run members:add -- ...` был отложен по запросу пользователя, то при фразе `код красный` нужно **сразу** продолжить именно эту отложенную команду.
- Команда запускается с теми же параметрами (`source-file`, `target-chat-id`, `start`, `count`, `invite-batch-size`), которые были отложены.
- Дополнительное подтверждение не требуется, если пользователь явно не отменял отложенную команду.
